Admin Screens — Platform Management

app/admin/home.tsx
🏠 Admin Dashboard
Main hub with user list, filtering by role/status, and inline approval actions. Skeleton loading, pull-to-refresh, error state.
  • Filter: ALL / STUDENT / TEACHER / ADMIN / PENDING / PENDING_AND_TEACHER
  • User list with localized role/status badges
  • Approve / reject pending registrations inline
  • Skeleton loading + pull-to-refresh
GET /api/v1/admin/users PUT /api/v1/admin/users/:id/approve
→ user-detail → broadcast → change-requests → settings → messages
app/admin/user-detail.tsx
👤 User Detail
Full profile for any user with admin actions: role change, status toggle, teacher assignment for students.
  • View full user profile
  • Activate / deactivate account
  • Assign / change teacher for students
  • Promote user to teacher role
GET /api/v1/admin/users/:id PUT /api/v1/admin/users/:id
app/admin/change-requests.tsx
🔄 Teacher Change Requests
Queue of student requests to change assigned teacher. Approve triggers teacher reassignment and FCM push.
  • Paginated request queue
  • Student + current teacher + requested teacher
  • Approve (reassigns) or Reject
  • FCM push notification on decision
GET /api/v1/teacher-changes PUT /api/v1/teacher-changes/:id
app/admin/broadcast.tsx
📢 Broadcast Message
Send a system message to all users or a specific role group.
  • Target: ALL / STUDENTS / TEACHERS
  • Compose message body
  • Send system announcement
POST /api/v1/messages/broadcast
app/admin/settings.tsx
⚙️ Settings
Profile management, password change, language toggle, and logout for admin.
  • Edit own profile
  • Change password
  • Language toggle (Arabic / English)
  • Logout
PUT /api/v1/users/profile PUT /api/v1/users/change-password
